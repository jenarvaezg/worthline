# Benchmark comparison is a present-time lens: two real-terms views globally, per-holding vs its tracked index

worthline shows how you have grown, but growth in a vacuum ("+12% en dos años") says nothing about whether that is _good_. The benchmark layer adds the verdict — am I keeping pace, or do I need to up my game? — and it is a **present-time, non-figure lens** in the same family as returns (ADR 0040), exposure profiles (ADR 0039/0058) and the contribution plan (ADR 0041): it is derived on read, never frozen into a snapshot, and never read by the net-worth / snapshot / ripple math (ADR 0008 untouched). The only thing persisted is the raw benchmark **series**, cached like a price — not a figure.

Benchmark series live in a **shared catalog in the control plane** (`benchmark_prices(series_id, date, value)`, one row per series per month), not per workspace: a series (Spanish CPI, a market index) is tenant-independent, so the fleet fetches it once — the same principle that moved exposure profiles to the control plane (ADR 0058). Cadence is **monthly** (CPI is monthly; it aligns with the monthly closes TWR rides in ADR 0040; daily would be overkill). Sources are **INE** (IPC general) for inflation and **Stooq** for index proxies, ingested by a **best-effort, isolated** phase on the daily-capture cron (ADR 0037 / #528) with **lazy idempotent backfill** — fetch only the missing months, first run fills the existing window — so a source outage never blocks snapshot capture.

**Global comparison — two real-terms lenses behind a toggle.** The naive "net-worth growth vs inflation" is misleading, because `Δ net worth = investment return + net contributions`: someone who only saves and never invests appears to "beat inflation" purely by adding money. So the global verdict is **two distinct, both-honest lenses**, selectable with a view toggle (RSC-first, ADR 0036), both expressed in **real terms** (deflated by CPI) and **annualized**:

- **Patrimonio (real)** — net worth deflated by CPI. Answers "is my wealth growing in purchasing power?"; it **includes contributions by design** and is labelled as such. **Ungated** (needs only existing snapshots + CPI) — the universal lens, and the one that ships first.
- **Rentabilidad (real)** — the invested sleeve's contribution-stripped return vs CPI. Answers "is my invested money outperforming inflation?". Scoped to operation-bearing holdings and **gated on the returns engine** (ADR 0040 / #547); the lens hides when there are no operations.

Separating the two into labelled lenses is what keeps each honest: the wealth question and the performance question are genuinely different, and conflating them (presenting contribution-driven net-worth growth as if it were investment performance) was the original error this ADR corrects.

**The performance lens uses an annualized, contribution-stripped measure — never the simple ratio.** It prefers the time-weighted rate (`annualizedRate`), then the money-weighted XIRR, falling back to CAGR. The simple total-gain ratio (gain ÷ total invested) is **rejected as a headline**: recent contributions inflate its denominator and can drag the verdict below inflation even when the money genuinely outperformed — the very contribution contamination this ADR exists to avoid. Cumulative TWR is likewise avoided as a headline, because small early balances can inflate it; the annualized figure is the robust one.

**Per-holding comparison** is a fund's time-weighted return (ADR 0040) against **the index it tracks**, keyed by the `tracked-index` label from the exposure catalog (ADR 0039/0058) → `series_id`, with total-return vs price-index honesty per instrument. It is **double-gated** on the returns engine (#547) and the exposure catalog (#539), and ships after the global lenses.

**Honesty and consequences.** Every benchmark figure is `null` with a stated reason when the series or the return is unavailable — never a fabricated line or number (the ADR 0040 signal pattern). Realizing the performance lens requires the returns engine to (a) expose the **monthly time-weighted series** (today only the endpoint rate is returned) and (b) compute returns **over an arbitrary window**, so the shared temporal filter can re-window the verdict; and the dashboard must feed `portfolioMonthlyCloses` into the portfolio return view (today it does not, so TWR is null on the home and the performance lens falls back to XIRR). These are the follow-on work items the S0 #621 prototype surfaced. This ADR is the benchmark decision that PRD #546 and issues #621–#626 reference (they cite a placeholder "ADR 0044", which is unrelated).
